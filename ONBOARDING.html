Strategic Partnerships Agents

Complete technical reference for the lead collection, scoring, outreach, and LinkedIn follow-up pipeline. Covers every parameter, data flow, and change guide a new student needs to operate and improve the system.

What This System Does

Collect

LinkedIn screenshots, post URLs, and manual contacts are parsed by GPT-4o Vision, deduped by domain + person, and written to a master CSV.

Rank

Each pending lead is scored 0–10 by GPT-4o against 5 criteria. Leads ≥ 5 enter the campaign shortlist (capped at 25). Leads < 5 go to backlog for next run.

Upload

After manual Apollo enrichment, the upload agent writes accounts and contacts into Notion with campaign sender and campaign ID tags.

Copywrite

4 messages per contact (LinkedIn cold, LinkedIn follow-up, email subject, email body) generated with A/B variant, sender context, trigger, and processed learnings.

LinkedIn Review

Saved connections HTML is parsed, matched to Notion contacts, and the agent drafts follow-ups or marks leads as ghosted based on configurable time windows.

Feedback Loop

Monthly: outcome classification → GPT-4o pattern analysis → learnings written to outreach_learnings.md → auto-injected into next copywriter run.

Campaign actions are manual and intentional. Only intake (collector) runs automatically via launchd. Ranking, outreach, and LinkedIn review happen on demand.

Full Data Flow

INPUT STREAMS data/inputs/images/new/ → collector → GPT-4o Vision extraction data/inputs/linkedin_urls/new/ → collector → web fetch + entity extraction data/inputs/manual_contacts/new/ → collector → direct parsing ↓ dedup by (domain, person_name) + per-company cap data/tables/master_input.csv ← single source of truth (11 cols, status=pending) ↓ ranking_agent → GPT-4o 0-10 score per lead ↓ score ≥ 5 ─→ up to 25 leads → weekly_qualified_leads_with_contacts.csv → weekly_qualified_leads_no_contact.csv → email report to team score ≥ 5 + over cap → backlog.csv (re-scored next run) score < 5 → backlog.csv (re-scored next run) ↓ [MANUAL] Apollo enrichment of shortlist CSVs ↓ upload_agent → Notion Accounts DB + Notion Contacts DB (dedup by Apollo ID / domain / name) ↓ copywriter_agent → 4 messages per contact written to Notion (LinkedIn cold + FU + email subject + body) A/B variant assigned per contact (50/50 random) ↓ [MANUAL] Send messages via LinkedIn / email ↓ linkedin_manager ← saved connections HTML (Cmd+S from LinkedIn) → match to Notion contacts → draft follow-ups (3-5 day window) → mark ghosted/nurture after configured window → update Notion status (never downgrade) ↓ [MONTHLY] feedback_agent → classify outcomes (success / fail / stale) → GPT-4o pattern analysis → outreach_learnings.md (auto-injected next copywrite)

Notion Status Hierarchy (never downgrade)

Status index ≥ 7 (Engaged and above) blocks a company from appearing on the campaign shortlist. The 16-level hierarchy is defined in agents/notion_cleanup.py (STATUS_HIERARCHY).

1 · New Lead 2 · Collected 3 · Qualified 4 · Apollo Enriched 5 · Outreach Ready 6 · Contacted 7 · Engaged ←block threshold 8 · Meeting Scheduled 9 · Meeting Done 10 · Proposal Sent 11 · Negotiating 12 · Partner Unqualified Nurture Ghost

Scoring Criteria (Ranking Agent)

GPT-4o receives the full SCORING_PROMPT (defined in ranking_agent.py) and returns a LeadScore Pydantic model: score (0–10 float), reasoning (str), red_flags (List[str]). The prompt is structured as follows:

1. Pipeline Success Context (dynamic, strongest signal)

Before each scoring run, build_pipeline_success_section() fetches companies from Notion Accounts that are in Workflow campaigns and have reached Engaged or further. These are injected into the prompt as proven lookalikes. Companies similar to these get +1 to +2 bonus mentioned in the prompt.

2. ICP — High Score (8–10)

  • Strong social, humanitarian, ecological, or innovation mission aligned with AI for Good
  • AI companies and tech scale-ups (Series A+) engaging with student ecosystems
  • Social impact organizations using or needing AI
  • Large corporates with AI initiatives and CSR programs
  • Innovation hubs, accelerators, tech ecosystems
  • NGOs, foundations, humanitarian organizations
  • Companies collaborating with student organizations (strong conversion signal)
  • VCs with explicit social impact / climate tech / sustainability focus

3. ICP — Medium Score (5–7)

  • Tech companies without clear AI focus but innovation-driven
  • Consulting firms with AI/tech practice
  • Research institutions and universities
  • Government digital/innovation agencies
  • Series A startups with clear AI application
  • Early-stage AI startups in accelerators/hackathons
  • General VCs with AI/tech focus but no explicit social/climate angle

4. ICP — Low Score (1–4)

  • Weak AI/impact connection
  • Traditional industries without innovation angle
  • B2C consumer companies without tech focus
  • Pure fintech/crypto VCs with no social angle

5. Automatic Disqualification (Score = 0)

These checks run before GPT-4o to save API costs. The two-step student club guard (rule-based → LLM verification) catches borderline cases.

CheckMethodExamples

A/B Testing (Copywriter Agent)

Each contact is randomly assigned variant A or B (50/50, random.choice(["A", "B"])). The variant is stored in Notion's AB Variant select property and persisted across runs.

Variant A — Concise Partner Mention

Framing injected into system prompt:

"We're developing AI solutions with partners like UN Women"

Understated. Let the partner name do the heavy lifting.

Variant B — Impact Scale Framing

Framing injected into system prompt:

"We're multiplying the impact of our non-profit partners like UN Women in over 50 countries through custom AI tools"

Emphasize scale and tangible reach. Make the reader feel it.

The feedback agent evaluates A/B results monthly: it classifies outcomes (success = status ≥ Engaged; failure = Unqualified or stale 30+ days), runs GPT-4o pattern analysis across both variants, and writes findings to data/prompts/outreach_learnings.md.

How to Make Changes or Improvements

This section is the guide for anyone who wants to tune the system. Always test with --dry-run first on a small batch.

Changing the scoring threshold or shortlist size

Open agents/ranking_agent.py. Find the two constants near line 799:

MIN_SCORE = 5.0       # leads below this go to backlog
MAX_QUALIFIED = 25    # shortlist cap per run

Lower MIN_SCORE to 4.0 if the pipeline is thin. Raise MAX_QUALIFIED if you want a larger Apollo batch. After changing, re-run python agent.py rank — existing leads in backlog will be re-scored.

Adding a company to the permanent blocklist

Open utils/config.py. Find COMPANY_BLOCKLIST and add the new string (lowercase, partial match is fine). This skips GPT-4o entirely for that company.

Adding a student club to the fast-filter

Open agents/ranking_agent.py. Add the exact name (lowercase) to STUDENT_CLUB_BLOCKLIST (line 68) or a pattern to STUDENT_CLUB_PATTERNS (line 92). If it's a common edge case, also add context to the SCORING_PROMPT examples section.

Changing scoring criteria or ICP definition

Edit SCORING_PROMPT in agents/ranking_agent.py (line 223). The prompt is plain text — no code changes needed. Key sections to modify:

  • HIGH/MEDIUM/LOW SCORE sections — add or remove bullet points
  • POSITIVE SIGNALS — add new boost signals
  • AUTOMATIC DISQUALIFICATION — add industry/type red flags

After editing the prompt, re-run ranking on a batch of 10–20 known leads and compare the new scores to previous ones to validate the change.

Changing the campaign cooldown window

Change CAMPAIGN_COOLDOWN_DAYS = 90 (line 334 in ranking_agent.py). This controls how long before a previously exported company can re-enter the shortlist. 90 days = ~3 months between campaigns on the same company.

Changing outreach message style or length

The full outreach brief lives in data/prompts/outreach_skill.md. Edit that file — no Python changes needed. Changes take effect immediately on the next copywrite run.

To change message length limits, edit the Pydantic field descriptions in class OutreachMessages (line 95, agents/copywriter_agent.py):

linkedin_first_cold: str = Field(description="... Max 75 words ...")
linkedin_follow_up:  str = Field(description="... Max 60 words ...")
cold_email_body:     str = Field(description="... Max 100 words ...")

Changing the A/B test variants

Edit VARIANT_ADDENDA dict in agents/copywriter_agent.py (line 79). Each key is "A" or "B", each value is a text block injected into the system prompt. You can add variant C by extending the dict and updating assign_variant() to use random.choice(["A", "B", "C"]).

Changing LinkedIn follow-up timing thresholds

Open agents/linkedin_manager.py. Look for the constants that define the follow-up window (3–5 days) and the ghost/nurture window. Update them and re-run with the saved HTML.

Adding a new input source to the collector

Open agents/collector.py. There are three clearly separated stream sections (screenshots, LinkedIn URLs, manual contacts). Add a new folder under data/inputs/, create a parser function, and register it in the main aggregation loop. Follow the existing pattern — each stream writes to the same 11-column master_input.csv schema defined in utils/config.py.

Adding a new Notion property to the upload

Open utils/notion_client.py. The upload_agent builds a properties dict before each Notion API call. Add your new field there. Then run python agent.py cleanup --domains to backfill existing records if needed.

Improving the feedback/learnings loop

Add real examples of weak vs. improved messages to the Notion Copywriter Improvement Log (toggle under Partnerships Agents page). Once you've collected 5+ good examples, run python agent.py feedback — it will distill new learnings into data/prompts/outreach_learnings.md. These are auto-injected the next time you run copywrite.

To manually add a learning without running the feedback agent, just append to data/prompts/outreach_learnings.md directly.

Automation Policy

Scheduled (launchd, runs automatically): collector (Mon/Wed/Fri 9am), notion_cleanup --all (1st & 15th 10am), feedback_agent (1st of month 11am), supervisor (Sat 9am).

On demand only: rank, upload, copywrite, linkedin.

Never run ranking_agent on a schedule. It emails the team and tags leads as qualified — this should always happen because someone deliberately asked for a new campaign, not on a timer.

API Cost Tracking

Every OpenAI call is logged to data/logs/api_usage.jsonl via utils/api_logger.py. The supervisor agent parses this file weekly and includes cost breakdown in the PDF audit report.

Copywriter Improvement Log

Use the Notion toggle log under the Partnerships Agents page. Each entry should include:

  • The weak snippet (bad example)
  • An improved version
  • The reason it was bad (too long, too generic, hallucinated fact, wrong tone, etc.)
  • Campaign and contact context

Once you have 3–5 examples, move processed learnings into data/prompts/outreach_learnings.md — either manually or by running python agent.py feedback. The copywriter auto-injects this file on the next run via load_learnings_prompt() in agents/copywriter_agent.py.
