Notion status hierarchy (16 levels)

Status index ≥ 7 (Engaged) blocks re-export. Defined in notion_cleanup.py → STATUS_HIERARCHY. Status is never downgraded.

Scoring Logic (ranking_agent)

GPT-4o receives SCORING_PROMPT and returns a Pydantic LeadScore: score (0–10 float), reasoning, red_flags. Prompt lives entirely in agents/ranking_agent.py — plain text, no recompile needed to edit.

Pipeline success context (strongest signal)

Before each run, build_pipeline_success_section() fetches Notion Accounts from Workflow campaigns at Engaged+ and injects them as lookalikes. Companies resembling these get a +1 to +2 prompt boost. As the pipeline matures this signal automatically improves.

A/B Test Variants

Assigned via random.choice(["A", "B"]) per contact, stored in Notion AB Variant property. The variant text is injected into the copywriter system prompt.

Variant A — Concise partner mention
"We're developing AI solutions with partners like UN Women"

Understated. Let the partner name carry the weight.

Variant B — Impact scale framing
"We're multiplying the impact of our non-profit partners like UN Women in over 50 countries through custom AI tools"

Scale and tangible reach. Make the reader feel it.

The feedback agent evaluates A/B results monthly, classifies outcomes (success = status ≥ Engaged; failure = Unqualified or stale 30+ days), and writes findings to data/prompts/outreach_learnings.md.

How to Make Changes

Always test with --dry-run first. For prompt changes, validate on a batch of 10–20 known leads and compare scores.

Change the score threshold or shortlist size

# agents/ranking_agent.py  ~line 799
MIN_SCORE = 5.0       # lower to 4.0 if pipeline is thin
MAX_QUALIFIED = 25    # raise if you want a larger Apollo batch

Add a company to the permanent blocklist

Open utils/config.py, add to COMPANY_BLOCKLIST (lowercase, partial match). GPT-4o call is skipped entirely for blocklisted companies.

Add a student club to the fast-filter

Add the name (lowercase) to STUDENT_CLUB_BLOCKLIST (line 68) or a pattern to STUDENT_CLUB_PATTERNS (line 92) in ranking_agent.py.

Change ICP definition or scoring criteria

Edit SCORING_PROMPT in ranking_agent.py (line 223). Plain text — no Python changes. Key sections: HIGH/MEDIUM/LOW SCORE, POSITIVE SIGNALS, AUTOMATIC DISQUALIFICATION.

Change the campaign cooldown

# agents/ranking_agent.py  ~line 334
CAMPAIGN_COOLDOWN_DAYS = 90  # days before same company can re-enter shortlist

Change outreach message style or length

Style lives in data/prompts/outreach_skill.md — edit the file directly, no Python needed. Length limits are in the Pydantic field description strings inside class OutreachMessages in copywriter_agent.py.

Change A/B variants

Edit VARIANT_ADDENDA dict in copywriter_agent.py (line 79). Add a variant C by extending the dict and changing assign_variant() to random.choice(["A","B","C"]).

Add a new input source

Add a folder under data/inputs/, write a parser function in collector.py, register it in the main aggregation loop. Each stream must produce rows matching the 11-column master CSV schema in utils/config.py.

Add a new Notion property

Edit the properties dict in utils/notion_client.py before the Notion API call. Re-run cleanup --domains to backfill existing records.

Improve the feedback / learnings loop

Add weak vs improved examples to the Notion Copywriter Improvement Log, then run python agent.py feedback. Or append directly to data/prompts/outreach_learnings.md — it's auto-injected on every copywrite run via load_learnings_prompt().
